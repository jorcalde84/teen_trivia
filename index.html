How do I set this up?

Pick the option that matches your comfort level. Option B is the most reliable — it works for any file size and avoids browser security blocks.

Option A: Drive share link Simple

  1. Upload questions.json to your Google Drive.
  2. Right-click → Share → set to "Anyone with the link".
  3. Copy the share link and paste it above.

⚠ Browsers may block this approach due to cross-origin restrictions. If it fails, use Option B.

Option B: Apps Script Web App Recommended

  1. Upload questions.json to Drive (no special sharing needed).
  2. Go to script.google.com → New project.
  3. Replace the default code with:
function doGet() {
  var FILE_ID = 'YOUR_FILE_ID_HERE';
  var content = DriveApp
    .getFileById(FILE_ID)
    .getBlob()
    .getDataAsString();
  return ContentService
    .createTextOutput(content)
    .setMimeType(ContentService.MimeType.JSON);
}
  1. Replace YOUR_FILE_ID_HERE with the file ID from your Drive link (the long string after /d/ in the URL).
  2. Click Save, then Deploy → New deployment.
  3. Set type to Web app, "Execute as: Me", "Who has access: Anyone".
  4. Click Deploy, copy the URL (ends with /exec), and paste it above.

Option C: Any direct JSON URL Flexible

Paste any URL that returns valid JSON — a GitHub Gist raw URL, your own server, etc. Same format as the export from this app.

⚠ Important

  • Sync replaces local edits. If you edit questions in this admin panel, export them and re-upload to Drive — otherwise your next sync will overwrite them.
  • The app caches the last-synced data, so it works offline after a successful sync.
